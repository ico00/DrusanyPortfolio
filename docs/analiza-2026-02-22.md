# Analiza projekta – 2026-02-22

Kompletna analiza DrusanyPortfolio projekta: sigurnost, duplikati, loše implementacije i predložene korekcije.

---

## 1. Sigurnost

### 1.1 sanitize.ts – whitelist i XSS

**Status:** Dobro, s manjim nedostacima

**Pronađeno:**
- `sanitizeProseHtml` koristi `sanitize-html` s whitelistom tagova i atributa
- `allowedSchemes` ograničen na `["http", "https", "mailto", "/"]`
- `transformTags` za `img` kopira sve potrebne `data-*` atribute

**Izvršena korekcija:**
- Dodan `transformTags` za `a` koji automatski dodaje `rel="noopener noreferrer"` na vanjske linkove (`http://`, `https://`) – zaštita od tabnabbing napada

### 1.2 API rute – path traversal, rate limit, validacija

**Status:** Dobro zaštićeno

**Pronađeno:**
- **Path traversal zaštita:**
  - `blog-delete-file/route.ts`: Provjerava `fullPath.startsWith(allowedDir + path.sep)`
  - `media-delete/route.ts`: Ista provjera
- **Rate limiting:**
  - Sve API rute koriste `checkRateLimit(request)` na početku
  - In-memory store: 200 req/min po IP
  - Napomena: Za produkciju s više instanci treba Redis (komentar već u `rateLimit.ts`)
- **Validacija:**
  - Upload rute provjeravaju `file.type`, `file.size`, magic bytes
  - `slug` i `date` se sanitiziraju (`sanitizeSlug`)

### 1.3 File upload – magic bytes, veličina, path traversal

**Status:** Dobro zaštićeno

**Pronađeno:**
- **Magic bytes:** `hasValidImageSignature()` provjerava JPEG, PNG, GIF, WebP (u `imageValidation.ts`)
- **Veličina:** `MAX_FILE_SIZE_BYTES = 20 * 1024 * 1024` (20 MB) u svim upload rutama
- **Path traversal:** Sanitizacija `slug` i `date` prije kreiranja putanje
- **sanitizeFilename():** Transliterira hrvatske znakove i uklanja posebne znakove

## 3. Loše implementacije (iz chat.md "loša faza")

### 3.1 ProseContent – useEffect vs useMemo, struktura

**Status:** Popravljeno

**Pronađeno:**
- `processProseHtml()` se koristi u `useMemo` – ispravno
- `useEffect` za keyboard events i body overflow – ispravno
- Struktura: HTML se obrađuje prije renderiranja s `linkedom` – ispravno

### 3.2 BlockNote – createPortal, MutationObserver

**Status:** Uklonjeno (iz chat.md Exchange 8)

**Pronađeno:**
- `FloatingBlockTypeBar.tsx` koristi siguran pristup (`position: fixed` + `getBoundingClientRect`)
- Nema `createPortal` ili `MutationObserver` u kodu

### 3.3 Sanitizer – svi potrebni atributi na whitelistu?

**Status:** Kompletan whitelist

**Pronađeno:**
- `PROSE_ALLOWED_ATTRIBUTES` uključuje sve potrebne atribute za `img`, `a`, `span`, `td`, `th`, `figure`, `div`
- `data-index` se postavlja na wrapper u `processProseHtml` (ProseContent), ne u sanitiziranom HTML-u – nije potreban u whitelistu

### 3.4 CSS – višak pravila, konflikti

**Pronađeno:**
- Stilovi za linkove (`.prose a`, `.prose-invert a`, `.blocknote-editor-wrapper a[href]`) – različiti konteksti, nije problem
- Višak `!important` u nekim pravilima – **predloženo:** Ukloniti gdje je moguće, koristiti specifičnije selektore

## 7. Zaključak

Projekt je dobro zaštićen (magic bytes, path traversal, rate limiting). Glavni problemi su:
1. **Dead code:** CustomLinkToolbar obrisan, exif-reader uklonjen
2. **Sigurnost:** Dodana zaštita od tabnabbing na vanjskim linkovima
3. **Duplikacija:** Sanitizacijske funkcije na više mjesta – moguća centralizacija
4. **CSS:** Mogućnost smanjenja `!important` i konsolidacije stilova

Većina problema je niskog prioriteta i ne utječe na funkcionalnost ili sigurnost.
